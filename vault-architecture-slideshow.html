Parsidion CC: The Obsidian-Powered Brain for Claude AI
user ~/Repos/parsidion-cc $ claude
A Claude Code customization toolkit that replaces built-in auto memory with an Obsidian vault-based knowledge management system, augmented by lifecycle hooks, a research agent, and graph-colorized vault exploration.
  • Automatic context loading at session start
  • Learning capture from every session transcript
  • AI-powered session summarization into structured notes
  • Working state snapshots before context compaction
  • Research agent that saves findings to the vault
Python 3.13+ stdlib only Obsidian config.yaml
System Architecture Overview
The system connects Claude Code's lifecycle events to an Obsidian vault through three hook scripts, a shared library, and an AI summarizer.
Four-Layer Architecture
Hooks → AI Summarizer → Shared Library → Obsidian Vault

Layer 1: Hook Scripts

Python scripts fired by Claude Code lifecycle events. Communicate via JSON stdin/stdout.

  • SessionStart — context injection
  • SessionEnd — learning capture
  • PreCompact — state snapshots

Layer 2: AI Summarizer

PEP 723 script using Claude Agent SDK. Processes pending sessions into structured vault notes.

  • Up to 5 parallel sessions
  • Reads pending_summaries.jsonl
  • Auto-launched by SessionEnd hook

Layer 3: vault_common.py

Shared library (stdlib-only). Frontmatter parsing, vault traversal, note search, path utilities.

  • Regex-based YAML parser
  • Config caching system
  • Git commit integration

Layer 4: ~/ClaudeVault/

The Obsidian vault. Auto-generated index, cross-linked notes, domain-based graph colors.

  • 9 organized subfolders
  • YAML frontmatter on every note
  • Wikilink cross-references

Session Lifecycle: Complete Data Flow
How knowledge flows through a full Claude Code session
1
SessionStart — Hook reads vault, injects project notes + recent notes as context
2
Session Active — Claude works with vault knowledge available in context window
3
PreCompact — If context fills up, snapshot current task + active files to daily note
4
Session Continues — Work resumes with compacted context + vault snapshot
5
SessionEnd — Analyze transcript, detect learnings, append to daily note
6
Queue — Significant sessions queued to pending_summaries.jsonl
7
Summarize — Background process generates structured notes via Agent SDK
8
Next Session — New notes are available for context injection on next start
A self-reinforcing loop: each session's learnings feed the next session's context

Trigger Evaluation: Skill Quality Assurance
Measure how accurately Claude invokes the vault skill
The SKILL.md description determines when Claude Code auto-invokes the vault skill. The trigger eval harness measures precision, recall, and accuracy to ensure the description is optimally tuned.

How It Works

1
Parse name & description from SKILL.md
2
20 test queries (10 should-trigger, 10 should-not)
3
Each query runs 3x for statistical reliability
4
6 parallel workers (ProcessPoolExecutor)
5
Computes precision, recall, accuracy

Key Details

  • 60 total API calls per eval run
  • 5 real distractor skills included for realism
  • Must run from separate terminal
  • Results saved to eval_results.json
bash ~/.claude/skills/\ claude-vault/scripts/\ run_trigger_eval.sh
What We Built

Core Components

  • SessionStart Hook — context injection from vault
  • SessionEnd Hook — learning capture & queuing
  • PreCompact Hook — state snapshots
  • Summarizer — AI-powered note generation
  • vault_common.py — shared stdlib-only library
  • update_index.py — auto-generated CLAUDE.md
  • Research Agent — vault-first web research

Design Principles

  • Python stdlib only — maximum portability
  • Obsidian-native — graph view, wikilinks, templates
  • Config-driven — centralized YAML with fallbacks
  • Git-integrated — optional auto-commit on writes
  • Self-reinforcing — each session feeds the next
  • AI-augmented — optional Haiku/Sonnet modes
