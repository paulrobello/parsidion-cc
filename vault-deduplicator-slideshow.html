Does vault-merge re-index?
A simple question reveals a missing feature
user → does vault-merge trigger re-index after a merge?
claude → grep -r "update_index" skills/parsidion/scripts/vault_merge.py
No matches found
No — vault-merge does not trigger a re-index after merging.
You'd need to run it manually:

$ uv run --no-project ~/.claude/skills/parsidion/scripts/update_index.py

Want me to add an automatic re-index call at the end of a successful merge?
user → yes, but add a --no-index option to disable it
Gap identified: batch workflows need --no-index to skip per-merge rebuilds and do one final rebuild at the end — much faster for 50+ merges.
