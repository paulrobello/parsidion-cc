Architecture Patterns

#architecture #vault #python

The vault uses a layered architecture with four main components: hook scripts, summarizer, shared library, and the vault filesystem itself.

Key Components

  • Hook scripts handle lifecycle events (session start/stop, compaction)
  • vault_common.py provides shared utilities
  • The summarizer processes queued sessions into vault notes

Final Design Summary

Layout

  • Resizable + collapsible left sidebar (browse-only file tree)
  • Note count per folder in tree
  • Tab bar for multiple open notes
  • Full-width content area (Read or Graph mode)
  • Read/Graph toggle in top-right toolbar
  • Single unified ⌘K search in toolbar

Search (⌘K)

  • Plain text → fuzzy title match
  • #tag → filter by tag
  • /folder → filter by folder
  • ⏎ opens in current tab
  • ⌘⏎ opens in new tab
  • In Graph mode: also flies to node

Graph Mode

  • Default: local 2-hop neighborhood
  • Config option for full-vault default
  • "Show Full Vault" button
  • Draggable HUD panel (existing controls)
  • Click node → opens note in tab
  • Existing physics simulation preserved

Interactions

  • Click wikilink → replace current tab
  • Cmd+click → new tab
  • Sidebar file click → open in current tab
  • ☰ toggle sidebar visibility
  • Drag sidebar edge to resize
  • All settings persisted to localStorage
