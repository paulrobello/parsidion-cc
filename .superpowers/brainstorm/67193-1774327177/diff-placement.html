Where should the git diff viewer live?

Three placement options for viewing note version history — each affects the workflow differently

A — Inline panel below note

History button in ReadingPane toolbar reveals a collapsible panel with commit list + diff below the note content. Everything stays in the same view.

B — Split-screen mode (replaces ReadingPane)

Clicking "History" replaces the reading pane with a two-column view: commits on the left, diff on the right. A back button returns to the note.

C — Floating modal / drawer

A slide-in drawer or modal overlays the current view. Note stays visible behind it. Good for quick checks without losing your place.
