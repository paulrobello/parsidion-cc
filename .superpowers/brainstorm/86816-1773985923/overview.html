Vault Visualizer — What We're Building

1,006 notes · 384-dim embeddings · Force-directed graph · Next.js

Concept Overview
Pre-computation (Python)
📦 Read embeddings.db
🔢 Compute pairwise cosine similarity for all 1,006 notes
✂️ Sparse export — keep only pairs above threshold
💾 Save as graph.json (nodes + edges)
Frontend (Next.js)
⚡ Load graph.json at startup
🕸️ D3 force simulation — nodes repel, edges attract
🎚️ Sliders: similarity threshold, repulsion, link strength
🖱️ Click node → full markdown note viewer
🔗 WikiLinks are clickable → jump to that node
🏷️ Filter by folder, tag, note type
🗂️ 1,006 notes 📐 384-dim BAAI/bge-small-en-v1.5 🔗 Wikilinks in related field 📁 9 folders

First question: Where does this app live?

The visualizer needs to read files from ~/ClaudeVault/ — where should it run?

A

Standalone local app

A separate Next.js project that you launch with bun dev and access at localhost. Reads graph.json generated by a Python script.

Simple · No auth needed · Direct file access
B

Inside parsidion repo

A visualizer/ subdirectory in this repo. Co-located with the Python scripts that generate the graph data.

One repo · Easy to update together · make visualizer
C

Served by parsidion-mcp

Extend the existing MCP server to serve the visualizer and expose the graph data via API endpoints. Always up, always fresh.

Live data · API-driven · More complex
